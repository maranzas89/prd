一、需求背景

现状问题

  • 机会分散 — Partner opportunities 通过邮件、Slack、电话零散进来，staff 用个人表格追踪，经常忘记哪些还有效。
  • 无 readiness 信号 — Counselor 没法一眼看出哪些学生真的到了推荐时机，判断全在脑子里，不在数据里。
  • referral 黑洞 — 学生被推荐后结果是黑洞，面了吗？拿 offer 了吗？没系统记录。
  • cohort 混乱 — 像 CHW + AlliedUP 这种深度合作需要一次匹配 20 个学生，今天只能靠表格、群邮件和祈祷。

为什么现在做

  • Calbright 的 "How It Works" 第 4 步就是 "Get the job you love"，career service 本来就是产品核心交付
  • 2026-01 CHW + SEIU-UHW + AlliedUP Apprenticeship 模型刚落地，给 L3 深度合作提供了成功模板
  • 现有 workforce partner 阵容已经就绪（Snap、Accenture、Docusign、Parker Dewey、Bay Area Council 等）
  • RIF 压力下，proving career outcomes 对学校预算叙事至关重要

五、详细方案

5.1 Partnership Hub - Three-Tier Model

  • 单页 + 3 Tab：L1 Internship / L2 Full-time / L3 Apprenticeship
  • 每个 Tab 共享同一套表头（Partner / Role / Program / Location / Deadline / Status），但 tier-specific 字段不同
  • Tab 切换保留筛选状态

5.2 Scenario 1: Partner Opportunity Intake

交互流程

flowchart TD A["Staff 点 New Opportunity"] --> B{"选 Tier"} B -->|L1| C1["L1: Duration + Stipend
+ Course-credit"] B -->|L2| C2["L2: Salary Range +
Employment Type + Start Date"] B -->|L3| C3["L3: Cohort Size + Wage +
WIOA funded + Sponsor + Commitment"] C1 --> D["填通用字段
Partner / Role / Program / Location / Deadline"] C2 --> D C3 --> D D --> E["保存"] E --> F["系统计算匹配学生池"] F --> G["出现在对应 Tab"]

规则

  • 必填字段缺失不让提交
  • L3 opportunity 创建后出现 "Start Cohort Matching" 按钮（L1/L2 不显示）
  • Partner contact email 存用于审计，不发自动邮件

5.3 Scenario 2: Job Readiness Score

公式

Score = Program Progress   × 30%
      + Workshop Completion  × 20%
      + CPL / Credential     × 15%
      + Intake Assessment    × 15%
      + Counselor Confidence × 20%

规则

  • Counselor Confidence 默认 null
  • 当 null 且 Program Progress ≥ 50% 时，学生 profile 显示 "Needs counselor input" banner
  • Score ≥ 70 → 绿色 "Ready" badge；50-69 → amber "Nearly Ready"；<50 → 无 badge
  • 每晚重算，也在关键事件触发时重算（workshop done / CPL approved / rating changed）
  • 保留 90 天趋势（sparkline）

5.4 Scenario 3: Matched Referral (L1/L2)

  • Career Partnership Card 显示最多 5 个匹配机会，按 match score 降序
  • 每张机会卡显示：Partner + Tier badge + Role + Match score + 2-3 条匹配原因 + Deadline 倒计时
  • 一键 Refer 开 modal：referral note（可选）+ 学生通知 toggle（默认 OFF）+ 确认
  • Override 路径：Browse all open opportunities → 选不匹配的 → 必填 20+ 字符 override 理由

5.5 Scenario 4: Referral Status Tracking

状态机

stateDiagram-v2 [*] --> Referred Referred --> PartnerReviewed : Partner opens PartnerReviewed --> InterviewScheduled : Schedule InterviewScheduled --> InterviewCompleted : Interview done InterviewCompleted --> OfferExtended : Partner offers OfferExtended --> OfferAccepted : Student accepts OfferExtended --> OfferDeclined : Student declines PartnerReviewed --> Rejected : Partner rejects InterviewCompleted --> Rejected : Partner rejects Referred --> Withdrawn : Withdrawn PartnerReviewed --> Withdrawn : Withdrawn InterviewScheduled --> Withdrawn : Withdrawn InterviewCompleted --> Withdrawn : Withdrawn OfferAccepted --> [*] OfferDeclined --> [*] Rejected --> [*] Withdrawn --> [*]

规则

  • 状态正向推进，允许回退但要填原因
  • Withdrawn / Rejected 是终止态
  • 无更新 > 14 天的 referral 自动浮到顶部，显示橙色 pin
  • Offer Extended / Offer Accepted 触发 staff 首页 banner + 可选邮件
  • 每条 referral 有 append-only 时间戳备注

5.6 Scenario 5: Cohort Matching (L3)

交互流程

flowchart TD A["L3 Opportunity"] --> B["Start Cohort Matching"] B --> C["候选池
按 program + readiness 过滤"] C --> D["按 Readiness Score 降序展示"] D --> E{"Staff 操作"} E -->|勾选/去选| F["手动调整"] E -->|Auto-fill to target| G["自动填满目标数"] F --> H{"非自动候选?"} G --> H H -->|是| I["填 20+ 字符说明"] H -->|否| J["锁定候选"] I --> J J --> K["Submit Cohort Referral
1 父 + N 子记录"]

规则

  • 锁机制：学生不能同时在两个 active cohort 里
  • 只有 cohort owner 或 admin 能解锁
  • cohort-level status rollup："Reviewed: 20 · Interviewed: 18 · Offered: 16 · Accepted: 14"
  • 终止态：Closed（全部学生 resolved）/ Abandoned（未响应取消）

5.7 Scenario 6: Partner Performance Dashboard

5 个核心 KPI

KPI 定义
Placement Rate接受 offer 数 / 总 referral 数
Offer Rate发 offer 数 / interview 数
Average Response Timereferred → reviewed 平均时长
Student Feedback1-5 分（毕业后调查）
Referrals Volume时间窗内 referral 总量

规则

  • 时间窗：30 / 90 / 365 / All Time（默认 90 天）
  • 低绩效阈值（只警告不排除）：Placement Rate < 20% / Response Time > 21 天 / Feedback < 3.0
  • 新 partner（< 5 次 referral close）显示 "New partnership — metrics available after first 5 referrals close"
  • Dashboard 顶部 tile：Top 5 by Placement Rate / Partners needing attention（3 worst）

七、异常与边界处理

场景 触发 系统行为 文案
学生同时被两个 cohort 选中 第二个 cohort 尝试 lock 阻止 + tooltip 说明 "Already locked in [Cohort Name] by [Staff]"
新 partner 无历史数据 Partner Performance 渲染 隐藏 KPI，显示提示 "New partnership — metrics available after first 5 referrals close"
Readiness Score 输入缺失 Canvas LMS 数据同步失败 保留旧 score + 显示警告 "Score based on cached data (last updated X hours ago)"
超过 14 天无状态更新 定时任务检测 橙色 pin + 浮到顶部 "Stale > 14 days — follow up?"
学生 withdrew from Calbright 学生状态变更 所有 active referral 自动 Withdrawn "Student withdrew — all referrals closed"
override referral 缺少理由 提交时 validation 不让提交 "Please explain why this non-matched referral is appropriate (min 20 chars)"
cohort 部分学生 decline Partner 反馈 cohort 不终止，partial rollup cohort 继续 active 到全部 resolved
Offer Accepted 但学生已 decline staff 误操作 二次确认 modal "Mark as Placed? This updates Partner Metrics."
staff 删除自己创建的 opportunity 已有 active referral 禁止硬删，改为 close + 保留历史 "Cannot delete — 3 active referrals exist. Close instead?"
