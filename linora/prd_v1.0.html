一、需求背景

现状问题

  • 求职者平均每天花 2-4 小时手动搜索、筛选和投递职位，效率极低
  • 简历与 JD 的匹配度分析完全依赖人工判断，缺乏量化评估
  • 简历撰写缺乏专业指导，关键词匹配不足导致 ATS 过滤率高
  • Onboarding 流程冗长，用户需在多个表单中重复填写相同信息
  • 现有求职工具（LinkedIn, Indeed）功能分散，无法提供一站式智能匹配

为什么现在做

  • Claude API 能力成熟，可准确解析简历并提供专业级优化建议
  • Supabase + Next.js 16 技术栈可快速构建全栈 MVP
  • 创始人 Wen 正在经历求职过程，对痛点有第一手深度理解

二、需求目标

七、异常与边界处理

场景 处理方式 用户提示
图片型 PDF 上传 pdf-parse 返回空文本，API 返回错误 「无法解析此 PDF，请上传文字版简历或手动填写」
网络断开 API 调用失败，catch 错误 「网络连接失败，请检查网络后重试」
Claude API 超时 设置 30s 超时，失败后允许手动输入 「AI 处理超时，您可以手动填写信息」
未登录访问受保护页面 Middleware 拦截，重定向到 /login 自动跳转到登录页
Onboarding 中途退出 onboarding_step 字段记录进度，下次登录恢复 「继续完成您的注册（第 X/8 步）」
积分不足 检查 credits 字段，不足时阻止 AI 调用 「积分不足，请充值后使用 AI 功能」
空数据状态 各页面显示空状态引导 「还没有数据，点击开始添加」
